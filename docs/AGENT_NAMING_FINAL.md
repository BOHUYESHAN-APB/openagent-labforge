# Agent 命名方案（最终版）

## 更新时间
2026-04-22

## 命名原则

1. **避免重复词汇**：不使用多个"执行器"、"编排器"
2. **功能清晰**：从名称就能看出用途
3. **分类明确**：按功能类型分组
4. **领域区分**：工程 vs 生信明确标识

## 精简模式（5-6 个 Agent）

适合新手和日常使用

| Agent | 中文名称 | 英文名称 | 核心功能 | 使用场景 |
|-------|---------|---------|---------|---------|
| sisyphus | 智能助手 | Smart Assistant | 最智能，自动判断和协调 | 不确定用什么时的默认选择 |
| prometheus | 任务规划 | Task Planner | 制定详细实施计划 | 复杂任务需要先规划 |
| executor | 计划路由 | Plan Router | 读取计划并智能路由 | 有计划后执行（/start-work） |
| wase | 工程全自动 | Engineering Auto | 工程任务全自动（内部规划+执行） | 工程类重型任务 |
| atlas | 快速开发 | Quick Dev | 轻量快速开发 | 简单快速的开发任务 |
| bio-autopilot | 生信全自动 | Bio Auto | 生信任务全自动 | 生信分析任务（默认启用）|

## 标准模式（额外 3 个 Agent）

适合中级和高级用户

| Agent | 中文名称 | 英文名称 | 核心功能 | 使用场景 |
|-------|---------|---------|---------|---------|
| orchestrator | 高级协调 | Advanced Coordinator | 高级编排能力 | 需要复杂协调的任务 |
| hephaestus | 深度开发 | Deep Dev | 深度分析和开发 | 需要深度思考的工程任务 |
| bio-pipeline-operator | 生信流程 | Bio Pipeline | 生信流程化任务 | 生信管道任务 |

## 命名分类

### 1. 通用类
- **sisyphus（智能助手）** - 最智能，什么都能做

### 2. 规划类
- **prometheus（任务规划）** - 制定计划
- **executor（计划路由）** - 执行计划（通过智能路由）

### 3. 全自动类
- **wase（工程全自动）** - 明确是工程类
- **bio-autopilot（生信全自动）** - 明确是生信类

### 4. 开发类
- **atlas（快速开发）** - 强调快速
- **hephaestus（深度开发）** - 强调深度

### 5. 专业类
- **orchestrator（高级协调）** - 高级编排
- **bio-pipeline-operator（生信流程）** - 生信流程

## 典型使用场景

### 场景 1：生信分析项目
```
用户: "帮我规划一个 RNA-seq 差异表达分析"
  ↓
选择: 任务规划（prometheus）
  ↓
生成详细计划
  ↓
用户: "/start-work"
  ↓
计划路由（executor）检测到生信任务
  ↓
路由到: 生信全自动（bio-autopilot）
  ↓
完成分析
```

### 场景 2：Web 应用开发
```
用户: "实现一个用户认证系统"
  ↓
选择: 工程全自动（wase）
  ↓
内部规划 + 执行
  ↓
完成前后端实现
```

### 场景 3：快速修复
```
用户: "修复登录页面的样式问题"
  ↓
选择: 快速开发（atlas）
  ↓
快速修复，无需规划
```

### 场景 4：复杂架构设计
```
用户: "重构整个数据库层"
  ↓
选择: 深度开发（hephaestus）
  ↓
深度分析 + 执行
```

## 命名优势

### 1. 避免重复
- ❌ 旧方案：主编排器、高级编排器（重复"编排器"）
- ❌ 旧方案：智能执行器、快速执行器、深度执行器（重复"执行器"）
- ✅ 新方案：智能助手、高级协调、快速开发、深度开发（无重复）

### 2. 分类清晰
- **助手类**：智能助手
- **规划类**：任务规划、计划路由
- **全自动类**：工程全自动、生信全自动
- **开发类**：快速开发、深度开发
- **专业类**：高级协调、生信流程

### 3. 领域明确
- **工程全自动** vs **生信全自动** - 清楚区分领域
- **快速开发** vs **深度开发** - 清楚区分深度
- **任务规划** vs **计划路由** - 清楚区分阶段

### 4. 功能直观
- "任务规划" - 一看就知道是做计划的
- "计划路由" - 一看就知道是路由计划的
- "快速开发" - 一看就知道是快速的
- "工程全自动" - 一看就知道是工程类全自动
